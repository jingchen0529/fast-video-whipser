# 真人短剧动作资产与版权区分方案

## 背景

当前系统已经在往“公共会话 + 视频分析 + 视频复刻”的方向演进。下一阶段如果要进一步提升内容复用能力和商业价值，一个很重要的方向是：

1. 从真人短剧里提取高价值的人物动作片段
2. 对这些片段做标签化、结构化沉淀
3. 在保留动作节奏、镜头语言、出场方式的前提下，生成“明显有区分的新版本”
4. 让这些片段逐步演化成可搜索、可复刻、可组合的动作资产库

这个方向既能服务内容分析，也能服务后续视频生成、视频复刻、动作模板推荐与版权风险控制。

## 目标

本方案的目标不是单纯做“视频剪辑”，而是构建一套围绕真人短剧的动作资产生产链：

- 能自动发现值得提取的角色动作片段
- 能把片段按“动作语义 + 角色气质 + 镜头语言 + 场景”打标签
- 能支持用自然语言搜索，例如“霸总走路”“强势出场”“回头冷看”
- 能基于原动作生成风格相近但视觉有明显区别的新片段
- 能对版权相似风险做基础评估，避免过于贴近原片

## 典型场景

### 场景一：动作资产沉淀

从真人短剧中提取下列片段并沉淀到资产库：

- 霸总走路
- 角色推门进入
- 回头停顿
- 女主受委屈抬眼
- 强势压迫式靠近
- 电梯门开后的出场

### 场景二：动作检索与复刻

用户输入：

> 找一些“霸总走路 + 办公室 + 压迫感出场”的片段

系统输出：

- 原始候选片段
- 标签解释
- 结构化动作摘要
- 可直接发起“换风格生成”

### 场景三：版权区分生成

用户选中某个片段后，生成一个“保留动作结构，但人物、空间、服装、镜头细节不同”的新片段，用于：

- 广告创意参考
- 短剧素材衍生
- 视频模板化生产
- 二次创作前的灵感生成

## 总体思路

建议把整条链路拆成 4 层：

1. 片段提取层
2. 标签理解层
3. 资产沉淀层
4. 版权区分生成层

### 1. 片段提取层

核心目标是从整段真人短剧里找出“值得存”的动作片段。

建议组合使用以下能力：

- 镜头切分
- 场景切分
- 人物检测与跟踪
- 姿态/动作识别
- 多模态语义总结

初期不必一步到位，可以采用“规则 + 多模态模型”混合策略。

### 2. 标签理解层

核心目标是把片段变成可搜索、可比较、可复用的数据对象。

至少要沉淀以下维度：

- 动作标签
- 情绪标签
- 出场方式标签
- 角色气质标签
- 场景标签
- 运镜标签
- 镜头景别标签

### 3. 资产沉淀层

核心目标是把原始视频切片沉淀成“可运营的动作资产”。

每个动作资产需要包含：

- 原始来源
- 起止时间
- 缩略图/预览
- 标签集合
- 结构化摘要
- 审核状态
- 是否允许继续衍生

### 4. 版权区分生成层

核心目标不是简单替换人物，而是在“动作保留”和“视觉区别”之间找到平衡。

要保留的东西：

- 节奏
- 动作骨架
- 出场逻辑
- 镜头推进关系

要主动改写的东西：

- 人物外观
- 服装造型
- 场景美术
- 光线与色调
- 摄影细节
- 局部动作微差

## 第一部分：片段提取方案

## 1. 提取目标

真人短剧中最值得提取的不是任意片段，而是“高可复用动作单元”。

建议优先抽取以下类别：

- 角色出场类
- 走路/靠近类
- 转身/回头类
- 停顿/凝视类
- 推门/开门进入类
- 对峙/压迫类
- 情绪爆发前兆类
- 受伤/脆弱姿态类

## 2. 提取流程

建议的基础流程如下：

1. 全片做镜头切分
2. 对每个镜头做人物检测和主角跟踪
3. 提取关键帧与短时动作特征
4. 用多模态模型判断该片段是否存在“明确动作语义”
5. 给出片段摘要与候选标签
6. 根据规则过滤无价值片段

## 3. 模型思路

### 方案 A：规则优先

适合 MVP 阶段，成本低、链路稳。

可依赖：

- 镜头切分
- 人物框/主角跟踪
- 姿态关键点
- 运动强度变化
- 场景切换

例如：

- 如果主角由远到近快速接近，同时镜头推近，则可能是“强势靠近”
- 如果人物停止移动后转头，看向镜头外对象，则可能是“回头停顿”

### 方案 B：多模态语义优先

适合中后期效果提升。

可以让模型对一个 2 到 6 秒的片段进行描述，例如：

> 男性角色从办公室门口推门进入，步伐平稳，先停顿半秒，再带压迫感向前走。

再把模型描述映射到标准标签体系。

### 建议

先做 A+B 混合：

- A 负责粗提取和过滤
- B 负责语义命名和补标签

这样更稳，也更容易控制成本。

## 2. 自由文本摘要

除了标准标签，每个片段还应该有一段人能直接理解的摘要，例如：

> 男主从电梯门打开后缓步走出，镜头跟拍，表情冷淡，停顿后转头看向走廊另一侧。

这段摘要对以下能力很重要：

- 语义搜索
- 审核人员理解
- 生成模型提示词构造

## 第三部分：动作资产库设计

## 1. 动作资产的核心对象

建议后续增加独立的动作资产表，例如：

- `motion_assets`
- `motion_asset_tags`
- `motion_asset_reviews`
- `asset_lineage`

每条动作资产建议包含：

- `id`
- `source_video_asset_id`
- `clip_asset_id`
- `start_ms`
- `end_ms`
- `action_summary`
- `action_label`
- `entrance_style`
- `emotion_label`
- `temperament_label`
- `scene_label`
- `camera_motion`
- `camera_shot`
- `review_status`
- `copyright_risk_level`
- `created_at`
- `updated_at`

## 2. 资产流程

建议动作资产有以下状态：

- `draft`
- `auto_tagged`
- `reviewing`
- `approved`
- `rejected`
- `derived`

典型流程：

1. 自动提取
2. 自动打标签
3. 人工审核
4. 批准入库
5. 支持搜索/推荐/复刻

## 3. 搜索方式

后续建议同时支持三种检索：

### 结构化筛选

例如：

- 角色气质 = `cold_dominant`
- 场景 = `office`
- 出场方式 = `door_entry`

### 关键词搜索

例如：

> 霸总走路

### 向量语义搜索

例如：

> 那种进门之后停一下再压迫式走过来的镜头

## 第四部分：版权区分生成思路

这是最关键也最敏感的一部分。

目标不是“洗稿”，而是“抽取动作结构并进行明显改写的再创作”。

## 1. 推荐的生成策略

建议把“动作保留”和“视觉重构”拆开理解。

### 保留层

保留以下内容：

- 角色运动轨迹
- 出场节奏
- 大体镜头运动
- 动作时序关系

### 改写层

主动改以下内容：

- 人物身份特征
- 发型/服装/体型
- 场景空间
- 光照与色彩
- 摄影细节
- 面部微表情
- 细节动作节奏

## 2. 具体生成方式建议

### 方案 A：基于原片段的受控视频变换

做法：

- 输入原片段
- 指定目标风格
- 指定“保留动作，不保留人物/场景”

优点：

- 结果容易保留动作感

缺点：

- 与原片容易过近
- 风险控制要求更高

### 方案 B：先抽动作，再重建视频

做法：

1. 从原片段抽出动作表示
2. 从动作表示和文本风格生成新片段

优点：

- 更利于版权区分
- 与原片视觉差异更大

缺点：

- 技术门槛更高
- 对动作保真要求更高

### 建议

中长期应优先靠近方案 B。  
短期 MVP 可以先从方案 A 起步，但必须加上风险控制。

## 3. 版权风险控制建议

后续建议加入一套“版权距离”评估机制。

至少从以下维度打分：

- 人物外观相似度
- 场景相似度
- 镜头结构相似度
- 动作时序相似度
- 构图相似度
- 关键帧视觉相似度

最后输出一个综合等级：

- `low`
- `medium`
- `high`

如果风险过高：

- 强制要求改写人物
- 强制要求改写场景
- 强制要求改写景别或运镜
- 强制增加动作微差

## 第五部分：产品能力设计建议

## 1. 在公共会话中的工作流

你当前产品很适合把这些能力都收进一条连续会话里。

建议工作流如下：

1. 上传一段真人短剧
2. 发起“动作片段提取”
3. 系统返回候选片段
4. 用户选中一个片段
5. 发起“换风格生成”
6. 系统返回多个区分版本
7. 用户继续追问或加入动作资产库

## 2. 建议新增的消息类型

- `motion_extract_request`
- `motion_extract_result`
- `motion_asset_card`
- `video_remake_request`
- `video_remake_result`
- `copyright_risk_report`

## 3. 建议新增的任务类型

- `motion_extraction`
- `motion_tagging`
- `motion_review`
- `video_remake`
- `copyright_distance_evaluation`

## 第六部分：推荐的分阶段实施路径

## Phase 1：先把提取链路打通

目标：

- 能从一段真人短剧里切出候选动作片段
- 能生成基础标签和摘要
- 能在会话中展示片段卡片

建议只做：

- 镜头切分
- 多模态描述
- 基础标签映射
- 候选片段展示

## Phase 2：建立动作资产库

目标：

- 可审核
- 可搜索
- 可复用

建议增加：

- `motion_assets` 表
- 审核状态
- 搜索接口
- 标签筛选

## Phase 3：做风格区分生成

目标：

- 基于片段生成明显不同的新版本
- 可以用于再创作参考或模板生产

建议增加：

- 复刻任务
- 风格参数
- 派生关系记录
- 版权风险评分

## Phase 4：做内容生产系统

目标：

- 动作资产推荐
- 一键生成系列短片
- 模板化生产

建议增加：

- 资产推荐
- 动作编排
- 多片段拼接
- 角色设定模板

## 第七部分：你当前最值得先做的事

结合你当前仓库状态，我建议优先级如下：

1. 先把“视频分析结果里返回候选动作片段”做出来
2. 再把片段卡升级成“动作资产候选卡”
3. 再加 `motion_assets` 数据表和搜索接口
4. 最后再做视频复刻与版权区分生成

原因很简单：

- 动作资产是底座
- 没有资产沉淀，后面的复刻和推荐都会很散
- 先把“提取 + 标签 + 入库”做扎实，后面扩展最稳

## 第八部分：一个建议的 MVP 范围

如果只做最小闭环，我建议 MVP 包含：

- 上传真人短剧
- 自动提取 3 到 10 个候选动作片段
- 自动打基础标签
- 在会话中展示片段卡
- 允许用户把片段加入动作资产库
- 支持关键词搜索动作资产

先不要在 MVP 阶段就追求：

- 全自动高精度动作理解
- 高质量视频重生成
- 复杂版权风险模型

先做可用，再做做强。

## 总结

这个方向本质上不是“做一个视频分析工具”，而是建立一套围绕真人短剧的动作知识库和生成链路。

最核心的思路可以总结成一句话：

> 先把真人短剧中的高价值动作片段提取并标签化，再基于这些动作资产做可控生成和版权区分。

如果这条线做顺了，后面你能做的不只是“分析视频”，而是：

- 动作资产库
- 剧情镜头模板库
- 风格复刻
- 角色动作推荐
- 批量内容生产

这会比单纯的视频下载或信息提取更有产品壁垒。
