## 一、推荐方案：两阶段 Pipeline（粗筛 + 精标）

### 为什么选这个方案

当前系统已经跑通了视频分析的完整链路：

```
视频/链接 → TikTok爬虫下载 → PySceneDetect镜头切分 → faster-whisper转写
→ LLM生成storyboard描述 → shot_segments表(start_ms/end_ms/transcript)
```

**推荐方案不是新建系统，而是在这条链路末尾接一步**：

```
shot_segments + storyboard描述 → 粗筛(规则过滤) → 精标(LLM打标签) → 写入motion_assets
```

### 方案优势

| 优势 | 说明 |
|------|------|
| **改动最小** | 不引入新的 CV 模型或视频处理库，复用 PySceneDetect + Storyboard 的已有输出 |
| **成本极低** | 粗筛零成本（纯文本匹配），精标只对候选片段调 LLM，单个短剧全流程 < ¥0.10 |
| **架构一致** | 沿用 WorkflowRegistry + TaskQueue + WorkflowEngine 的已有模式，新建一个独立 Workflow |
| **立即可用** | `motion_assets` 表已存在，`/api/assets/motions` 接口已有骨架，补逻辑即可 |
| **精度可迭代** | 粗筛关键词表和精标 Prompt 都是配置，调优不改架构 |
| **冷启动快** | 用 TikTok 爬虫批量导入短剧 → 全自动跑提取流程 → 资产库有内容 |

### 两个阶段做什么

**第一阶段：粗筛（零成本，高召回）**

- 输入：storyboard 的镜头描述文本 + shot_segment 的时长信息
- 逻辑：关键词匹配 + 时长过滤
- 输出：候选 shot_segment_id 列表
- 成本：纯文本处理，< 1 秒，无 API 调用

```python
# 粗筛核心逻辑
MOTION_KEYWORDS = {
    "walk_in": ["走进", "走入", "步入", "进来", "walk in", "stride"],
    "push_door": ["推门", "开门", "push door", "open door"],
    "turn_back": ["回头", "转身", "turn", "look back"],
    "approach": ["靠近", "走向", "逼近", "approach", "walk toward"],
    "stare": ["凝视", "注视", "盯", "stare", "gaze"],
    "sit_down": ["坐下", "落座", "sit down"],
    "stand_up": ["站起", "起身", "stand up"],
    "slap": ["扇", "打耳光", "slap"],
    "embrace": ["拥抱", "抱住", "embrace", "hug"],
    "collapse": ["瘫坐", "倒下", "崩溃", "collapse"],
    "throw": ["扔", "摔", "砸", "throw", "slam"],
    "kneel": ["跪", "kneel"],
}

def coarse_filter(shot_segment, storyboard_text):
    """
    粗筛：基于时长 + 关键词过滤。
    返回 (是否候选, 匹配到的标签列表)
    """
    duration = shot_segment["end_ms"] - shot_segment["start_ms"]
    # 时长过滤：太短(<1s)或太长(>8s)的片段价值低
    if duration < 1000 or duration > 8000:
        return False, []

    text = (storyboard_text or "").lower()
    matched = []
    for label, keywords in MOTION_KEYWORDS.items():
        if any(kw in text for kw in keywords):
            matched.append(label)

    return len(matched) > 0, matched
```

一个 3 分钟短剧切出约 30 个镜头，粗筛后通常剩 5-8 个候选。

**第二阶段：精标（低成本，高精度）**

- 输入：候选片段的 storyboard 描述 + transcript + 时长
- 逻辑：调用 LLM（GPT-4o / 当前已接入的模型），输出结构化标签 JSON
- 输出：完整的标签集 + 摘要 + 置信度
- 成本：5-8 个候选 ≈ \$0.03-0.05

## 二、新建独立 Workflow：`motion_extraction`

### 为什么不是在 analysis workflow 后面加步骤

虽然追加 Step 10 技术上可行，但我建议**新建独立的 Workflow**，原因：

1. **解耦**：动作提取是独立业务，不应该和视频分析强绑定。用户可能只想分析不想提取，也可能对已分析的视频事后再提取
2. **可重跑**：独立 workflow 可以对同一个视频反复执行（换 Prompt、换规则后重新提取），而 analysis workflow 重跑成本更高
3. **新菜单入口**：你要新的菜单和工作流，独立 workflow 对应独立页面最清晰
4. **前置依赖明确**：motion_extraction workflow 需要 analysis workflow 的输出（shot_segments + storyboard），但不需要强耦合在一个流程里

### Workflow 定义

新建文件 `app/workflows/motion_extraction.py`：

```python
"""
Motion Extraction workflow: extract and tag motion assets from analyzed videos.
Requires: analysis workflow must have completed for the target project.
"""
import logging
from typing import Any

from app.workflows.engine import (
    TaskStepDefinition,
    WorkflowContext,
    WorkflowDefinition,
    WorkflowRegistry,
)

logger = logging.getLogger(__name__)

MOTION_EXTRACTION_STEPS = (
    TaskStepDefinition(
        "validate_analysis_data",
        "检查分析数据",
        "确认目标视频已完成分析，shot_segments 和 storyboard 数据可用。",
    ),
    TaskStepDefinition(
        "coarse_filter_shots",
        "粗筛候选片段",
        "基于关键词和时长规则，从所有镜头中筛选可能包含高价值动作的片段。",
    ),
    TaskStepDefinition(
        "tag_candidates",
        "精标动作标签",
        "对候选片段调用多模态模型，生成结构化标签和动作摘要。",
    ),
    TaskStepDefinition(
        "generate_thumbnails",
        "生成预览缩略图",
        "为每个通过精标的动作片段提取关键帧作为缩略图。",
    ),
    TaskStepDefinition(
        "save_motion_assets",
        "写入动作资产",
        "将标签化的片段写入 motion_assets 表，状态设为 auto_tagged。",
    ),
    TaskStepDefinition(
        "finish",
        "提取完成",
        "动作资产提取流程全部完成，可在资产库中查看。",
    ),
)


@WorkflowRegistry.register
class MotionExtractionWorkflow(WorkflowDefinition):
    """6-step motion asset extraction workflow."""

    workflow_type = "motion_extraction"

    def __init__(self):
        super().__init__()
        self._setup_steps()

    def _setup_steps(self) -> None:
        from app.services.motion_service import MotionService
        self._service = MotionService()

        steps_config = [
            (MOTION_EXTRACTION_STEPS[0], self._validate_analysis_data, True),
            (MOTION_EXTRACTION_STEPS[1], self._coarse_filter_shots, True),
            (MOTION_EXTRACTION_STEPS[2], self._tag_candidates, True),
            (MOTION_EXTRACTION_STEPS[3], self._generate_thumbnails, True),
            (MOTION_EXTRACTION_STEPS[4], self._save_motion_assets, True),
            (MOTION_EXTRACTION_STEPS[5], self._finish, True),
        ]
        for definition, handler, append_msg in steps_config:
            self.register_step(definition, handler, append_status_message=append_msg)

    async def _validate_analysis_data(self, ctx: WorkflowContext) -> dict[str, Any]:
        return await self._service.step_validate_analysis(project_id=ctx.project_id)

    async def _coarse_filter_shots(self, ctx: WorkflowContext) -> dict[str, Any]:
        return await self._service.step_coarse_filter(project_id=ctx.project_id)

    async def _tag_candidates(self, ctx: WorkflowContext) -> dict[str, Any]:
        context = {"coarse_filter_shots": ctx["coarse_filter_shots"]}
        return await self._service.step_tag_candidates(
            project_id=ctx.project_id, context=context
        )

    async def _generate_thumbnails(self, ctx: WorkflowContext) -> dict[str, Any]:
        context = {"tag_candidates": ctx["tag_candidates"]}
        return await self._service.step_generate_thumbnails(
            project_id=ctx.project_id, context=context
        )

    async def _save_motion_assets(self, ctx: WorkflowContext) -> dict[str, Any]:
        context = {
            "tag_candidates": ctx["tag_candidates"],
            "generate_thumbnails": ctx["generate_thumbnails"],
        }
        return await self._service.step_save_assets(
            project_id=ctx.project_id, context=context
        )

    async def _finish(self, ctx: WorkflowContext) -> dict[str, Any]:
        return await self._service.step_finish(project_id=ctx.project_id)
```

### 各步骤详细逻辑

#### Step 1: validate_analysis_data

```python
async def step_validate_analysis(self, project_id: int) -> dict:
    """检查 shot_segments 和 storyboard 是否存在"""
    db = create_connection()
    shots = db.execute(
        "SELECT COUNT(*) as cnt FROM shot_segments WHERE project_id = ?",
        (project_id,)
    ).fetchone()

    if shots["cnt"] == 0:
        raise WorkflowStepError("该视频尚未完成分析，请先执行视频分析流程。")

    storyboards = db.execute(
        "SELECT COUNT(*) as cnt FROM storyboard_items WHERE storyboard_id IN "
        "(SELECT id FROM storyboards WHERE project_id = ?)",
        (project_id,)
    ).fetchone()

    return {
        "detail": f"数据就绪：{shots['cnt']} 个镜头片段，{storyboards['cnt']} 条分镜描述。",
        "shot_count": shots["cnt"],
        "storyboard_count": storyboards["cnt"],
    }
```

#### Step 2: coarse_filter_shots

```python
async def step_coarse_filter(self, project_id: int) -> dict:
    """粗筛：关键词 + 时长规则"""
    db = create_connection()

    # 查所有 shot_segments，关联 storyboard 描述
    shots = db.execute("""
        SELECT s.id, s.start_ms, s.end_ms, s.transcript_text,
               si.visual_description, si.narration_text
        FROM shot_segments s
        LEFT JOIN storyboard_items si ON si.shot_segment_id = s.id
        WHERE s.project_id = ?
        ORDER BY s.start_ms
    """, (project_id,)).fetchall()

    candidates = []
    for shot in shots:
        # 合并所有文本信息用于匹配
        combined_text = " ".join(filter(None, [
            shot["transcript_text"],
            shot["visual_description"],
            shot["narration_text"],
        ]))
        is_candidate, matched_labels = coarse_filter(shot, combined_text)
        if is_candidate:
            candidates.append({
                "shot_segment_id": shot["id"],
                "start_ms": shot["start_ms"],
                "end_ms": shot["end_ms"],
                "matched_labels": matched_labels,
                "combined_text": combined_text,
            })

    return {
        "detail": f"粗筛完成：从 {len(shots)} 个镜头中筛出 {len(candidates)} 个候选。",
        "total_shots": len(shots),
        "candidate_count": len(candidates),
        "candidates": candidates,
    }
```

#### Step 3: tag_candidates

```python
async def step_tag_candidates(self, project_id: int, context: dict) -> dict:
    """精标：调 LLM 给候选片段打结构化标签"""
    candidates = context["coarse_filter_shots"]["candidates"]
    tagged_results = []

    for candidate in candidates:
        duration_sec = (candidate["end_ms"] - candidate["start_ms"]) / 1000.0
        prompt = TAGGING_PROMPT.format(
            duration_sec=f"{duration_sec:.1f}",
            transcript=candidate.get("combined_text", "无"),
            storyboard_desc=candidate.get("combined_text", "无"),
        )

        # 调用已有的 LLM 接口
        response = await self._llm_client.chat(prompt)
        tags = parse_json_response(response)

        if tags and tags.get("is_high_value", False) and tags.get("confidence", 0) >= 0.6:
            tagged_results.append({
                **candidate,
                "tags": tags,
            })

    return {
        "detail": f"精标完成：{len(tagged_results)} 个高价值动作片段。",
        "tagged_count": len(tagged_results),
        "tagged_results": tagged_results,
    }
```

```python
async def step_save_assets(self, project_id: int, context: dict) -> dict:
    """写入 motion_assets 表"""
    tagged = context["tag_candidates"]["tagged_results"]
    thumbnails = context["generate_thumbnails"]["thumbnails"]

    db = create_connection()
    saved_ids = []

    for item in tagged:
        tags = item["tags"]
        asset_id = str(uuid.uuid4())
        now = utcnow_iso()

        db.execute("""
            INSERT INTO motion_assets (
                id, source_video_asset_id, start_ms, end_ms,
                action_summary, action_label, entrance_style,
                emotion_label, temperament_label, scene_label,
                camera_motion, camera_shot,
                origin, review_status, copyright_risk_level,
                metadata_json, created_at, updated_at
            ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, 'ai_generated', 'auto_tagged', 'unknown', ?, ?, ?)
        """, (
            asset_id,
            get_video_asset_id(project_id),  # 关联源视频
            item["start_ms"],
            item["end_ms"],
            tags.get("action_summary", ""),
            tags.get("action_label"),
            tags.get("entrance_style"),
            tags.get("emotion_label"),
            tags.get("temperament_label"),
            tags.get("scene_label"),
            tags.get("camera_motion"),
            tags.get("camera_shot"),
            json.dumps({
                "confidence": tags.get("confidence", 0),
                "matched_labels": item.get("matched_labels", []),
                "thumbnail_path": thumbnails.get(item["shot_segment_id"]),
            }),
            now, now,
        ))
        saved_ids.append(asset_id)

    db.commit()
    return {
        "detail": f"已保存 {len(saved_ids)} 个动作资产到资产库。",
        "saved_count": len(saved_ids),
        "asset_ids": saved_ids,
    }
```

## 四点五、与现有资产体系的关系

动作资产提取是独立板块，核心职责是沉淀可搜索、可审核、可复用的动作资产，不要求与 `remake` 板块形成业务闭环。

### 4.5.1 数据关系

建议统一为下面的关系：

1. 原始参考视频
   - 落在 `media_assets`
2. 分析输出的镜头片段
   - 落在 `shot_segments`
3. 从镜头片段中提取出的动作标签
   - 落在 `motion_assets`
4. 其他板块产生的视频结果
   - 也可以落在 `media_assets`

这样就能形成：

```text
原始视频(media_assets)
  └── 分析结果(shot_segments/storyboards)
        └── 动作资产(motion_assets)
```

### 4.5.2 关键约束

1. 动作资产本身不是成片存储表，动作提取板块的主落点是 `motion_assets`。
2. `motion_assets` 主要存标签、摘要、片段区间和来源关系。
3. 如未来其他板块需要引用动作资产，建议通过来源 ID 或 metadata 做弱关联。
4. 前端“动作资产库”页面只需保证动作资产自身可查、可筛、可审核。

### 4.5.3 推荐扩展字段

当前不一定要马上改表，但开发文档建议预留下面这些信息到 `metadata_json`：

```json
{
  "confidence": 0.92,
  "matched_labels": ["push_door", "walk_in"],
  "thumbnail_path": "/tmp/thumb.jpg",
  "project_id": 123,
  "source_shot_segment_id": "shot_xxx",
  "usage_count": 0,
  "external_refs": []
}
```

## 五点五、状态机与审核流转

为了避免“候选结果”和“正式资产”混在一起，建议明确状态机。

### 5.5.1 `motion_assets.review_status`

建议统一使用以下状态：

1. `draft`
   - 手工创建但未完成标注
2. `auto_tagged`
   - 自动提取完成，待人工审核
3. `approved`
   - 已确认可进入动作资产库正式使用
4. `rejected`
   - 审核不通过
5. `archived`
   - 保留历史但不再推荐

### 5.5.2 推荐操作规则

1. 自动提取写入时默认 `auto_tagged`
2. 用户点击“加入资产库”后变更为 `approved`
3. 被判定质量差或不适合入库的片段改为 `rejected`
4. 长期不用或被替代的资产可改为 `archived`

### 5.5.3 列表页默认展示规则

1. 动作提取工作台展示：
   - `auto_tagged`
   - `approved`
2. 动作资产库默认展示：
   - `approved`
3. 管理员模式下可切换查看：
   - `auto_tagged`
   - `rejected`
   - `archived`
